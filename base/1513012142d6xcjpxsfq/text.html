Branch

Создаем новую ветку и переключаемся на нее

git checkout -b <branch-name>

Alternatives:

git branch <branch-name> && git checkout <branch-name>

Создаем новую ветку без родительской ветки

git checkout --orphan <branch-name>

Как быстро переключиться на предыдущую ветку

Список локальных и удаленных веток

Выводим список веток в удаленном репозитории

Выводим все ветки, включая удаленные, а так же последний комит в них

Как переименовать ветку

git branch -m <new-branch-name>

Alternatives:

git branch -m [<old-branch-name>] <new-branch-name>

Как удалить локальную ветку

git branch -d <local-branch-name>

Как удалить ветку в удаленном репозитории

git push origin --delete <remote-branch-name>

Alternatives:

git push origin :<remote-branch-name>

Как показать название текущей ветки

git rev-parse --abbrev-ref HEAD

Показать все ветки, не слитые в master

git checkout master && git branch --no-merged

Выводим список веток, которые уже слиты с веткой master

git branch --merged master

Перемещаем ветку feature на master и мержим ее в мастер

git rebase master feature && git checkout master && git merge -

Удаляем ветки, которые уже слиты с master

git branch --merged master | grep -v '^\*' | xargs -n 1 git branch -d

Alternatives:

git branch --merged master | grep -v '^\*\| master' | xargs -n 1 git branch -d #

will not delete master if master is not checked out

Находим ветки, которые содержат комит с указанным хешем

git branch -a --contains <commit-ish>

Alternatives:

git branch --contains <commit-ish>

Track upstream branch

git branch -u origin/mybranch

Commit

Меняем сообщение последнего комита

Выполняем команду, открывается редактор, указанный в настройках git. Меняем текст сообщения, сохраняем файл и закрываем редактор.

Сообщение можно указать и непосредственно при вызове команды с помощью опции

-m (--message)

git commit --amend


# можно указать сообщение с помощью опции -m

git commit --amend -m "New message"

Изменить предыдущий коммит без изменения сообщения к коммиту

git commit --amend --no-edit

Как поменять имя автора последнего комита

git commit --amend --no-edit --author='Author Name <email@address.com>'

Сбросить автора, после того как автор был изменен в глобальной конфигурации

git commit --amend --reset-author --no-edit

Создать комит только с указанными файлами

git commit --only <file_path>

Создаем комит, обходя хуки pre-commit и commit-msg

Alternatives:

Как отметить комит как исправление к указанному комиту

git commit --fixup <SHA-1>

Config

Показать конфиг и все псевдонимы (alias)

Как изменить локальный/глобальный конфиг git

git config [--global] --edit

Изменить текстовый редактор

git config --global core.editor '$EDITOR'

Игнорировать изменения прав доступа к файлам при коммите

git config core.fileMode false

Сделать git чувствительным к регистру

git config --global core.ignorecase false

Включить автоматическое исправление опечаток

git config --global help.autocorrect 1

Отключить цветной вывод Git

git config --global color.ui false

Настройки цвета

git config --global <specific command e.g branch, diff> <true, false or always>

Удалить запись из глобального конфига

git config --global --unset <entry-name>

Повторное использование записанных разрешений, запись и повторное использование разрешений предыдущих конфликтов

git config --global rerere.enabled 1

Всегда выполнять перемещение вместо слияния при получении изменений из удаленного репозитория

git config --global pull.rebase true

Alternatives:

#git < 1.7.9

git config --global branch.autosetuprebase always

Псевдонимы (alias) для команд Git

git config --global alias.<handle> <command>

git config --global alias.st status

Merge

Слить ветку feature с master, объединив все коммиты ветки feature в один

При этом коммит слияния не будет создан, вам нужно будет сделать его вручную.

git merge feature --squash
