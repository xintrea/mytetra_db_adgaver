Введение

Программные продукты Microinvest рассчитаны на работу в сети. В отличие от SQL сервера, с его легкой и логической настройкой (указывается IP адрес SQL сервера), в базе Access необходимо провести несколько предварительных настроек на уровне операционной системы. Настройки могут быть осуществлены стандартным пользователем с основными знаниями операционной системы.

Основные понятия

База Access представляет собой один общий файл, который содержит все данные фирмы. Этот файл открывается программой Microinvest, что позволяет сохранять операции и изготавливать отчеты. Работу в сети, необходимо организовать таким образом, чтобы файл был доступен всем рабочим местам, и были разрешены операции «Просмотр» и «Запись».
В работе с базой Access можно выделить два типа компьютеров:

  • Основной компьютер. Он исполняет функции сервера и сохраняет файл с базой данными;
  • Дополнительные компьютеры. Это отдаленные рабочие места (компьютеры), которые работают с базой данных на основном компьютере.

Настройка основного компьютера

Основной компьютер содержит файл с базой данными. Чтобы эта база была доступна другим компьютерам необходимо пройти определенные шаги:

  1. Файл с базой данными сохраняется в отдельной, самостоятельной папке. Лучше всего, чтобы папка находилась в основном каталоге диска и была предназначена только для базы с данными в сети;
  2. Эта папка должна быть видна в сети для каждого компьютера. В операционной системе эта функция называется “Share” (папка с общим доступом), которая позволяет делать папку доступной в сети;
  3. При указании общего доступа необходимо дать полные права всем пользователям. Это необходимо, так как пользователи будут просматривать и записывать операции;
  4. Папку необходимо наименовать. Возможно, чтобы имя совпадало с реальным именем диска.

Когда все вышеперечисленные условия исполнены, то база с данными становится доступна другим компьютерам. Сейчас необходимо настроить отдаленные рабочие места.

Настройка отдаленных компьютеров

Отдаленные рабочие места являются «клиентами» основного компьютера. Настройка всех отдаленных компьютеров идентична и содержите следующую последовательность:

  1. Необходимо зайти в центральный компьютер по сети. Должны появиться все папки с общим доступом (одна или несколько);
  2. Выбираем папку, которая содержит базу с данными и присваиваем сетевой диск к ней. В операционной системе это называется Подключение сетевого диска (Map Network Drive). После исполнения этой операции в системе появляется новый диск, который содержит папку с базой и саму базу с данными;
  3. В программном продукте указывается этот диск и из него выбирается база с данными.

Когда все рабочие места открыли одну и ту же базу с данными, то они начинают работать вместе и в сети. Это технология, когда все выполняется в реальном времени и каждый отдаленный компьютер имеет доступ к операциям других операторов.

Особенности

При работе с базой Access в сети, необходимо учитывать некоторые особенности:

  1. Основной компьютер необходимо запускать первым, потому что на нем хранится файл с базой данными. Если это не соблюдено, то база не будет доступна отдаленным компьютерам, и они перейдут в автономный режим работы;
  2. При выполнении процедуры Map Network Drive, желательно установить настройку „Reconnect at logon“ (Восстанавливать при входе в систему), чтобы настройка автоматически восстанавливалась после перезапуска компьютера. Если это не исполнено, необходимо будет каждый раз настраивать компьютер;
  3. Очень важно настроить правильно права для общего доступа. Если они не настроены, то после запуска выходит ошибка в программе (даже если база видна в сети);
  4. Данная процедура относится ко всем программным продуктам Microinvest;
  5. Данная процедура предназначена для работы в локальной сети. Сеть через Internet настраивается сложно и не является надежной при работе с базой данными такого типа. Так как вся процедура концентрируется на файле с общим доступом, то дополнительные настройки должны обеспечивать стабильную и быструю работу с этим файлом.
