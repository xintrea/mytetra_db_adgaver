Происходит, когда поступает новый элемент в папке "Входящие".

Синтаксис

выражение . NewMailEx ( EntryIDCollection )

выражение Переменная, которая содержит объект приложения .

Параметры

ИмяОбязательный или необязательныйТип данныхОписание
EntryIDCollectionОбязательноеStringСтрока, представляющая идентификатор записи элемента, полученных в папке "Входящие".

Заметки

Это событие вызывается, когда для каждого элемента, полученных, который обрабатывается в Microsoft Outlook. Элемент может иметь одно из нескольких различных типов элементов, например, MailItem , MeetingItem или SharingItem . Строка EntryIDsCollection содержит идентификатор записи, соответствующий этому элементу. Обратите внимание, что это поведение изменилось из более ранних версий событий, когда EntryIDCollection содержится список разделенных запятой идентификаторы всех элементов, полученных в папке "Входящие" после последнего выполнения событие.

Это событие не доступны в Microsoft Visual Basic Scripting Edition (VBScript).

Это событие вызывается для учетных записей электронной почты, которые обеспечивают уведомления для сообщений, полученных, такие как учетные записи POP3 и Microsoft Exchange Server.

Событие NewMailEx вызывается при получении нового сообщения в папке "Входящие", прежде чем правила клиента обрабатывается. Идентификатор записи, возвращаемых в массиве EntryIDCollection можно использовать для вызова метода NameSpace.GetItemFromID и обработки элемента. Используйте этот метод с осторожностью для сведения к минимуму влияние на производительность Outlook. Тем не менее, в зависимости от программы установки на стороне клиента компьютера, после прибытия нового сообщения в папке "Входящие" обрабатывает как фильтрации нежелательной почты и правил клиента, переместите нового сообщения из папки «Входящие» в другую папку может возникнуть асинхронно. Не следует предполагать, что после эти события возникают, это всегда будет один элемент увеличение числа элементов в папке "Входящие".

Для пользователей с учетной записью Exchange Server (не - режима кэширования данных Exchange или режима кэширования данных Exchange) событие будет выполняться только для сообщений, приходящих на сервере после запуска Outlook. Событие не будет срабатывать, для сообщений, которые синхронизируются в режиме кэширования Exchange сразу после запуска Outlook, а также для сообщений, которые уже на сервере при запуске Outlook в не - режима кэширования данных Exchange.

Для пользователей с помощью режима кэширования данных Exchange, события будут срабатывать в все параметры, предоставленные, что запущен Outlook при получении сообщения: Загружать сообщения целиком, Загружать заголовкии Загружать заголовки, а затем сообщения.
