Пример организации интерфейса с оригинальными возможностями

Автор: Зубков Евгений zevplus@gmail.com

Данный пример может служить основой для разработки интерфейса приложения на MSA.
Особенности:
• Outlook подобное представление информационного пространства как три зоны, разделенные двумя «разделителями».
• Хранение параметров формы в отдельной таблице с привязкой некоторых из них к текущему пользователю.
• Изменение содержания навигационной панели в зависимости от выбранной «левой» кнопки.

Этот «кирпичик» как и все последующие, написан с использованием кодов взятых из интернета.
У меня есть две просьбы к читателям:
• Присылать варианты, оптимизирующие предлагаемые процессы (снижающие размер кода и увеличивающие быстродействие).
• Присылать идеи – как можно создать интерфейс подобный Outlook 2010.

Пример написан в MSA 2003, но работает и на MSA XP

Учебная база данных содержит 2 урока:

Первый урок
- демонстрирует основные шаги по разработке интерфейса универсальной панели (УП). УП разработана исходя из традиционных представлений о трех оконном интерфейсе:
• Дерево навигации;
• Отфильтрованный Список объектов;
• Детальная информация о выбранном объекте (или статистическая информация о группе отфильтрованных объектов)
