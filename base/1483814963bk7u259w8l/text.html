Что это за зверь?
PyCharm - интегрированная среда разработки для языка программирования Python. Предоставляет средства для анализа кода, графический отладчик, инструмент для запуска юнит-тестов и поддерживает веб-разработку на Django. PyCharm разработана чешской компанией JetBrains на основе IntelliJ IDEA.PyCharm работает под операционными системами Windows, Mac OS X и Linux.
(википедия)

Плюсы
+ прекрасное автодополнение кода( в том числе django-шаблоны, на порядок лучше аналогов, так же хорошее автодоплнение js, css, html и т.д.)
+ хорошие возможности для отладки приложения
+ удобная работа с серверами контроля версии (СКВ)
+ возможность работы с базами данных( так же хорошее автодоплнение SQL и много другое)
+ возможность работы с кодом находящимся на удалённой машине( так же его отлаживание)
+ исправление зависимостей автоматическое
+ встроенный shell
+ ...
Плюсов очень много, перечислять все не буду, можно зайти на сайт и почитать, для навичков главное, удобая отладка из коробки, удобная работа с СКВ( из коробки), хорошее автодополнение.

Минусы
- платная( есть бесплатная, но она обрезаная)
- написана IDE на java, а главная проблема java? - Правильно, есть кучу ресурсов. Если оставить IDE на долгое время в режиме отладки, может все повиснуть или очень сильно тупить...
- порой чудит при отладке( бывают разные мелки баги)
- нету русской версии( для новичков бывает страшно)

Вообщем и целом прекрасная IDE для работы на python.

Начало работы

  1. Создание проекта.


    2. Вот такое вас встретить, когда загрузите PyCharm3. Слево - список ваших проектов, справо что-то типо мини-меню. Предлагаю потом провести настройки, в начале создадим проект

    1-я строка - название проекта
    2-я строка - его место расположение
    3-я строка - тип проекта, он вполне обширен
    4-я строка - интерпретатор питона, вот тут и идём к следующему пункту( не пытайтесь создать проект без указания интерпретатора)

  2. Настройка вирутального окружения для python

Займёмся настройка интепритатора python. Если у вас уже установлен питон на машине, то почти 100%, что pycharm найдёт его( если указан в переменных пути), если нет, то придётся заняться этим ручками. Жмём на кнопочку, где три точечки( или Shift+Enter), что бы открыть настройки интепритатора.

Ну справо видим три кнопочки. 
1) Добавить интепритатор( если вы уже создали виртуальное окружение или хотите помочь pycharm найти ваш дефолтный интерпретатор или вообще подключиться к удалённому интепритатору python).
2) Удаление интепритатора из настроек(!!)
3) Создание виртуального окружения.

Важно: необходимо указать в 3-ей строке интепритатор python для создания virtualenv.

Давайте создаим django-проект.

Дальше выскочит:

Тут 1-я строка название вашего проекта, дальше идёт название вашего приложения, затем папка для хранения шаблонов. В самом внизю включение админки в проект( иначе, строки с админкой будут закоментированы). Pycharm создат вам проект, для той версии django, которая у вас стоит, на данный момент у меня стояла 1.3.7( поэтому такая странная структура проекта)
