Настройка git для работы через proxy

Понадобилось настроить git для работы через proxy-сервер. Кроме того, в силу особенностей нашего прокси, приходится отключать SSL.

Т.к. у нас используется доменная авторизация, то я обладаю следующими исходными данными:

  • имя пользователя — domain\username
  • пароль — password
  • адрес proxy-сервера — proxy.domain.org
  • порт — 3128

Таким образом, в настроечный файл git’a (он находится в ~/.gitconfig) добавить следующий код:

[http]

proxy = http://domain\\username:password@proxy.domain.org:3128

sslVerify = false


Либо сделать это через командную строку:


3user@host:~$ git config --global http.proxy http://domain\\username:password@proxy.domain.org:3128

 

user@host:~$ git config --global http.sslVerify false

В данном случае я делаю настройки глобальные (ключ —global). Если нужно настроить такие параметры только для какого-то одного репозитария — то команду выполнять надо из директории репозитария без ключа —global.
